准备工作

在开始之前，确保你手头有：

项目 要求
服务器 一台能访问 Anthropic API 的 VPS（境外或有出口代理）
系统 Ubuntu 20.04 / 22.04 / Debian 11+ 均可
权限 root 或具备 sudo 权限的账户
网络 服务器能正常 curl https://api.anthropic.com
域名（可选） 如需 HTTPS 访问，准备一个解析好的域名

⚠ 注意： Anthropic API 在中国大陆无法直连，请确保服务器在境外（如香港、美国、日本等节点）。

申请 Anthropic API Key

第一步：注册 Anthropic 账号

访问 https://console.anthropic.com，使用邮箱注册账号并完成邮件验证。

第二步：充值额度

Anthropic 采用预付费模式，新账号需要先充值才能调用 API：

  1. 登录后进入控制台 → Billing → Add credits
  2. 最低充值 $5，支持信用卡/借记卡
  3. 充值完成后自动获得对应 token 额度

💡 小提示： 新账号注册后可能有少量免费额度用于测试，但正式使用仍需充值。

第三步：生成 API Key

  1. 进入 API Keys 页面
  2. 点击 Create Key 按钮
  3. 输入一个便于识别的名称，如 openclaw-server
  4. 点击确认，立即复制并保存显示的 Key（格式：sk-ant-api03-xxxxxxxx）

⚠ 重要： API Key 只在创建时显示一次，请立刻保存到安全的地方（如密码管理器）。

常见报错与解决

401 Unauthorized

原因： API Key 填写有误，或 Key 前后有多余空格。

解决： 重新检查 .env 中的 ANTHROPIC_API_KEY，确保无空格、无换行。

403 Forbidden / permission_error

原因： 账户未充值或额度耗尽。

解决： 登录 console.anthropic.com → Billing，检查余额并充值。

connection timeout / curl: (28)

原因： 服务器无法访问 Anthropic API（被墙或网络问题）。

解决：

  1. 确认服务器在境外节点
  2. 检查服务器防火墙是否拦截了出站 443 端口
  3. 可临时测试：curl -v https://api.anthropic.com --max-time 10

npm install 卡住不动

原因： npm 默认源在境外，速度慢。

流式输出中断 / 打字卡顿

原因： Nginx 默认缓冲导致 SSE 流被截断。

解决： 检查 Nginx 配置中是否已加入 proxy_buffering off;（参考上方反向代理配置）。

总结

完成本教程后，你已经：

  • ✅ 申请了 Anthropic API Key
  • ✅ 在服务器上部署了 OpenCLAW 面板
  • ✅ 成功接入 Claude API 并完成对话验证
  • ✅ （可选）配置了 Nginx + HTTPS 反向代理

  • 在 .env 中添加更多模型，按场景切换
