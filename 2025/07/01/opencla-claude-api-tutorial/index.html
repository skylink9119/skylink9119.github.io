本文按 2026 年 3 月 26 日可查到的 OpenClaw 官方文档思路整理，目标只有一个：让你从零把 OpenClaw 跑起来，并且能切换接入 OpenAI、Anthropic Claude、Google Gemini、OpenRouter 等主流 AI 厂商接口。

OpenClaw 是什么

OpenClaw 可以理解成一个自托管 AI 网关和控制面板。它的核心作用不是“单独提供模型”，而是把你自己的模型 API Key 接进来，再通过一个统一的界面和工作流去使用。

对新手来说，它有几个明显优点：

  • 统一管理多个厂商 API，不用每次换平台都重新折腾
  • 可以在同一个环境里切换模型
  • 支持本地或服务器部署
  • 后续扩展聊天、网关和自动化能力更方便

如果你以前只用过网页版 ChatGPT、Claude 或 Gemini，那么可以把 OpenClaw 理解为“你自己的 AI 控制台”。

这篇教程适合谁

这篇文章适合：

  • 第一次接触 OpenClaw
  • 会基础 SSH，但没独立部署过 AI 面板
  • 想一套环境同时接入多家模型
  • 想把 API Key 放在自己的机器或 VPS 上管理

如果你完全不会 Linux，也没关系，照着命令一步一步执行就可以。

部署前准备

建议你先准备好下面这些：

项目 建议
系统 macOS / Linux / Windows（有 WSL 也可以）
服务器 推荐 2 核 2G 以上 VPS，系统建议 Ubuntu 22.04+/Debian 12+
Node 环境 按官方思路，建议 Node.js 20 或更高
网络 服务器需要能访问对应厂商 API
域名 非必须；如果你要公网长期使用，建议配域名和 HTTPS
API Key 至少准备一家，推荐先准备 OpenAI 或 OpenRouter

建议新手的最低组合：

  • 一台 Ubuntu 服务器
  • 一个 OpenAI 或 OpenRouter Key
  • 一个能 SSH 登录服务器的终端工具

OpenClaw 的核心思路

很多人第一次接触时最容易混淆一件事：

OpenClaw 本身不是模型厂商，它更像一个统一接入层。

也就是说：

  • OpenClaw 负责把界面、配置、路由和调用管理起来
  • 真正回答问题的，是你接入的 OpenAI、Claude、Gemini、OpenRouter 等模型

所以你后面要做的事情，本质上就是把不同厂商的 Key 填进去，再选默认模型。

其他厂商怎么接

除了上面几家，很多厂商都提供了 OpenAI-compatible 接口。常见思路是：

  • 准备对应平台的 API Key
  • 找到它的 Base URL
  • 在 OpenClaw 中按兼容 OpenAI 的方式配置

这一类通常包括：

  • DeepSeek
  • Groq
  • SiliconFlow
  • Together AI
  • 火山方舟
  • Azure OpenAI

如果某个平台文档写明“兼容 OpenAI API”，你就重点看两样：

  • API Key
  • Base URL

然后在 OpenClaw 里把它当成兼容提供商接入即可。

推荐的新手接入顺序

如果你是第一次搭，建议不要一上来就接 5 家。

最稳妥的顺序是：

  1. 先装好 OpenClaw 并确认控制台能打开
  2. 先接 OpenAI 或 OpenRouter，确保第一条对话成功
  3. 再接 Claude
  4. 最后再接 Gemini 和其他兼容厂商

这样做的好处是，一旦报错，你更容易知道问题出在哪一层：

  • 是 OpenClaw 没装好
  • 是网络问题
  • 是 Key 错了
  • 是某家厂商的接口配置不对

我给新手的实际建议

如果你问我，第一次搭 OpenClaw 最推荐的路线是什么，我会建议：

  1. 先在一台干净的 Ubuntu 服务器上安装 OpenClaw
  2. 先只接 OpenRouter
  3. 跑通第一条对话
  4. 再单独补接 OpenAI、Claude、Gemini
  5. 最后再做反向代理、HTTPS、长期运行优化

这样成功率最高，也最不容易被一堆配置同时绊住。

总结

现在你只要记住一件事就够了：

OpenClaw 负责“统一管理”，真正提供能力的是你接入的各家模型 API。

所以最合理的搭建思路就是：

  • 先把 OpenClaw 装好
  • 再接一家最容易成功的 API
  • 跑通后再逐步增加厂商

如果你照着这篇做，最少可以完成这三件事：

  • 成功安装并打开 OpenClaw
  • 接入至少一家主流 AI 厂商 API
  • 后续继续扩展到 OpenAI、Claude、Gemini、OpenRouter 等多模型环境
