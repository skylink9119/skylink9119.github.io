OpenClaw 的 AGENTS.md、SOUL.md、USER.md 应该怎么写详细教程

发表于2026-03-27|更新于2026-03-27|AI工具
|浏览量:
OpenClaw Series

这篇文章属于 OpenClaw 系列教程

如果你还想连同安装、助理设定、配置模板和内容自动化一起系统看完，可以直接进入专题导航页；如果你更喜欢视频讲解，也可以跳去 YouTube 频道。

去 YouTube 看演示 查看 OpenClaw 专题

很多人知道 OpenClaw 支持 AGENTS.md、SOUL.md、USER.md，但真正开始写时就卡住了。原因通常不是不会写字，而是不知道这三个文件到底分别负责什么。写混了，效果就会很差。

先说结论：这三个文件分别是干什么的

你可以这样理解：

  • AGENTS.md：规定它怎么做事
  • SOUL.md：规定它像什么样的人
  • USER.md：规定你是谁、你要什么

如果你还是分不清，我给你一个最简单的判断方法：

  • 这是工作规则吗？放 AGENTS.md
  • 这是性格和表达风格吗？放 SOUL.md
  • 这是用户画像和长期偏好吗？放 USER.md

这三个文件并不是越长越好，而是越清楚越好。

先理解一个原则：不要把三个文件写成一锅粥

最常见的错误就是：

  • 在 SOUL.md 里写工作流程
  • 在 AGENTS.md 里写用户目标
  • 在 USER.md 里写一堆人格设定

结果就是 OpenClaw 读进去了，但行为并不稳定。

所以你写之前，先记住一个原则：

每个文件只承担一种职责。

AGENTS.md 应该怎么写

AGENTS.md 最适合放这些内容：

  • 角色职责
  • 工作原则
  • 优先级
  • 输出规则
  • 文件组织规则
  • 敏感信息规则

你可以把它理解成“助理工作手册”。

一个适合大多数人的结构

AGENTS.md 的关键点

你写它时，要重点回答这些问题：

  • 它主要负责什么
  • 遇到任务时优先考虑什么
  • 输出应该长什么样
  • 什么内容要记住
  • 什么内容不能乱写

SOUL.md 应该怎么写

SOUL.md 不负责流程，它负责气质。

你可以理解为：

  • 助理的人设
  • 表达习惯
  • 沟通方式
  • 价值取向

例如：

SOUL.md 最适合写什么

  • 它说话是强势还是温和
  • 它是偏专业、偏生活化，还是偏创作型
  • 它是否需要更强调效率、执行力、商业化意识

SOUL.md 不适合写什么

  • 文件路径规则
  • 记忆写入规则
  • 项目目录结构
  • 技术性约束

这些都应该放回 AGENTS.md。

USER.md 应该怎么写

USER.md 是让 OpenClaw 理解“你”的地方。

这里最适合写：

  • 你的身份
  • 你的主要工作内容
  • 你的长期目标
  • 你的风格偏好
  • 你的做事习惯

例如：

USER.md 的核心价值

它能让 OpenClaw 明白：

  • 你做什么行业
  • 你最关心什么结果
  • 你更喜欢怎样的输出

这会明显减少“每次都从零解释自己”的成本。

这三个文件怎么配合

最容易理解的方式是：

  • AGENTS.md 负责“规则”
  • SOUL.md 负责“气质”
  • USER.md 负责“用户画像”

如果把它们放在一起看，大概就是：

只要这三者分工明确，OpenClaw 的稳定性就会好很多。

一个适合内容创作者的完整示例

如果你主要做博客和 YouTube，其实可以直接用这种思路。

AGENTS.md

写：

  • 优先输出可执行内容
  • 优先服务博客和 YouTube 工作流
  • 长期信息写入 MEMORY.md

SOUL.md

写：

  • 简洁
  • 可靠
  • 不空话
  • 强执行感

USER.md

写：

  • 你是技术内容创作者
  • 主线是 VPS、AI 工具、教程
  • 博客的目标是承接搜索流量
  • YouTube 的目标是承接视频转化

这套组合就已经很实用了。

写这三个文件时最容易踩的坑

1. 写得太长

内容太长不一定更好，反而可能稀释重点。

2. 写得太空

比如“请你成为最好的助理”，这种几乎没用。

3. 重复

同一件事在三个文件里都写一遍，会让边界模糊。

4. 经常改来改去

如果规则还没跑够几轮就频繁改，很难判断哪里有效。

5. 只写一次就不维护

随着你工作方向变化，这三个文件也应该逐步更新。

最推荐的写法策略

如果你现在还没有这些文件，我建议按这个顺序写：

  1. 先写 USER.md
  2. 再写 AGENTS.md
  3. 最后写 SOUL.md

原因很简单：

  • 先知道你是谁
  • 再决定它怎么帮你
  • 最后决定它以什么风格帮你

这样逻辑最顺。

总结

OpenClaw 里的这三个文件，其实并不复杂。

真正关键的是分工：

  • AGENTS.md 管规则
  • SOUL.md 管风格
  • USER.md 管用户画像

只要你把边界分清楚，再持续微调，OpenClaw 就会越来越像一个真正理解你、而不是每次都随机发挥的长期助理。
