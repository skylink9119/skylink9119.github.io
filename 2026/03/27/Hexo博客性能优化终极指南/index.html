本文记录了我对个人Hexo博客进行的全面性能优化过程，涵盖了从基础配置到高级优化的完整方案。优化后，博客加载速度从原来的3-5秒降低到1秒以内。

📊 优化前性能基准

在开始优化前，先用 Lighthouse 测试了博客性能：

  • Performance: 65/100
  • First Contentful Paint: 2.8s
  • Largest Contentful Paint: 4.2s
  • Total Blocking Time: 280ms
  • Cumulative Layout Shift: 0.15
  • 博客体积: 4.3MB

🎯 优化目标

  1. Performance评分 > 90/100
  2. 首屏加载时间 < 1s
  3. 博客体积减少50%+
  4. 移动端体验优化

1. 图片优化（最快见效）

问题分析

  • 未压缩的图片占用了大量带宽
  • 缺少现代图片格式支持（WebP）
  • 没有懒加载机制

解决方案

优化效果

  • JPEG图片质量从100%降到80%（肉眼几乎无差别）
  • PNG图片使用有损压缩
  • 图片体积平均减少60%
