如果你用了 profile，要特别注意

官方文档特别提醒了一个容易漏掉的点：

如果你使用过 --profile 或 OPENCLAW_PROFILE，状态目录和服务名可能都不是默认值。

比如可能会变成：

  • ~/.openclaw-<profile>
  • ai.openclaw.<profile>
  • openclaw-gateway-<profile>.service
  • OpenClaw Gateway (<profile>)

这种情况下，你要把默认名字替换成你自己的 profile 名，再删一次对应文件和服务。

如果你是从源码运行的

有些人不是通过安装脚本或全局 npm 装的，而是：

然后在仓库里直接跑。

这种情况官方建议顺序是：

  1. 先卸载网关服务
  2. 再删除源码仓库目录
  3. 再删除状态目录和工作区

也就是说，不要先把仓库删了，结果服务还在后台跑。

怎么确认已经卸载干净了

你可以按下面这个清单检查：

  • openclaw 命令已经不可用，或者版本命令找不到
  • 本地没有 OpenClaw 网关服务在跑
  • ~/.openclaw 或对应 profile 目录已经删除
  • 工作区目录已经删除
  • macOS 的 LaunchAgents / Linux 的 systemd user unit / Windows 的计划任务已经删除
  • 如果安装过 Mac App，/Applications/OpenClaw.app 也已经删掉

最后给普通用户的建议

如果你只是本地电脑不用 OpenClaw 了，我建议你按这个顺序做：

  1. 先 openclaw backup create
  2. 再 openclaw uninstall
  3. 如果还有残留，再按平台手动清理
  4. 最后再检查状态目录、服务和 CLI 是否还存在

这是最稳、最不容易误删其他文件的方式。

总结

OpenClaw 官方目前给出的卸载路径其实很清楚：

  • 优先用 openclaw uninstall
  • 卸不干净再手动删服务、状态目录、工作区和 CLI
  • 如果用了 profile 或自定义配置路径，要额外检查对应目录
