去 YouTube 看演示 查看 OpenClaw 专题

本文基于 2026 年 3 月 26 日可查到的 OpenClaw 官方安装文档整理。当前官方建议环境是 Node 24，Node 22.16+ 仍然兼容；而在 Windows 上，官方明确更推荐使用 WSL2 安装 OpenClaw。

先说最推荐的安装方式

如果你只想知道“我到底该怎么装最稳”，那答案是：

  • macOS：直接用官方安装脚本
  • Windows：优先用 WSL2 + Ubuntu 安装

虽然 OpenClaw 也提供了 Windows PowerShell 安装脚本，但按官方文档目前的建议，Windows 原生环境不如 WSL2 稳定。

所以这篇文章我会分成两部分：

  1. macOS 安装 OpenClaw
  2. Windows 安装 OpenClaw

另外我也会顺手写上“安装后怎么验证是否成功”。

安装前你需要准备什么

建议先确认下面这些条件：

项目 建议
网络 能访问 openclaw.ai、npm 和对应模型厂商 API
Node.js 官方当前推荐 Node 24，最低兼容 Node 22.16+
包管理器 默认走 npm 即可，pnpm 适合有经验用户
磁盘空间 至少预留 2GB 以上，避免后续构建或缓存报错
权限 本地电脑建议使用当前用户安装，不建议乱用 root/管理员装一切

如果你是纯新手，我建议：

  • macOS：直接跑官方脚本
  • Windows：先安装 WSL2，再在 Ubuntu 里跑官方脚本

安装成功后建议马上做的事

装完不要立刻关终端，建议先做这几步：

  1. 跑一次 openclaw doctor
  2. 确认 openclaw status 正常
  3. 打开 openclaw dashboard
  4. 先做一次最基础的本地对话测试
  5. 再去接 OpenAI、Claude、Gemini 或 OpenRouter

这样你后面如果出问题，会更容易判断到底是“安装问题”还是“模型 API 接入问题”。
