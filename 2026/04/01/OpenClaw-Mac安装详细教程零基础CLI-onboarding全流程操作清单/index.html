OpenClaw Mac 安装教程：macOS 零基础从安装到跑通 Dashboard

OpenClaw Series

这篇文章属于 OpenClaw 的 macOS 新手安装细化篇

如果你想看最细的 Mac 安装步骤，这篇最适合；如果你想回到整个系列的总入口，可以先去专题页，再按自己的阶段继续往下看。

查看 OpenClaw 专题 返回文章归档

本文按 2026 年 4 月 1 日可查到的 OpenClaw 官方 Getting Started、Onboarding Wizard、Dashboard、FAQ 与 doctor 文档整理。由于不同版本的界面文字可能略有变化，本文采用“看见什么就选什么”的方式，尽量让零基础新手也能直接照着做。

先说结论：Mac 新手第一次安装，最稳的路线只有这一条

如果你是第一次接触 OpenClaw，不要一开始就研究 Docker、源码安装、Telegram、Tailscale 或公网访问。

最稳的顺序是：

  1. 用官方安装脚本安装 OpenClaw
  2. 运行 openclaw onboard --install-daemon
  3. 按向导把本地 Gateway 配好
  4. 用 openclaw gateway status 确认服务正常
  5. 用 openclaw dashboard 打开浏览器并完成第一次聊天

先把这 5 步跑通，你后面再接手机聊天渠道、做远程访问、配技能和自动化，排错难度都会低很多。

这篇教程适合谁

这篇文章特别适合下面这几种情况：

  • 你用的是 macOS
  • 你几乎没有命令行经验
  • 你想按官方推荐方式先把 OpenClaw 装起来
  • 你不想一上来就接 Telegram、Discord 或 WhatsApp
  • 你想要的是“每一步点什么、输什么、看到什么算正常”

如果你现在的目标只是“先在这台 Mac 上把浏览器聊天跑通”，那这篇就是最适合你的版本。

开始前要准备什么

正式开始前，你只需要准备 3 样东西：

1. 一台能正常联网的 Mac

  • 建议你使用自己的管理员账号
  • 安装过程中可能会弹出系统权限确认
  • 网络需要能访问 OpenClaw 官方安装脚本和你使用的模型服务

2. 一个可用的模型 API Key

OpenClaw 本身不是模型提供商，它只是 Gateway 和控制层。
所以你至少要准备一种可用的模型接入方式。

对零基础新手来说，第一次最省事的是：

  • 你已经有 OpenAI API Key
  • 或者你已经有 Anthropic API Key

如果你没有 API Key，也不是完全不能安装，但最后大概率会卡在“界面能打开，却聊不起来”。

3. 一个打开终端的方法

按下面操作：

  1. 键盘按 Command + Space
  2. 输入 Terminal
  3. 回车打开

后面的命令都在这个窗口里执行。

这一步正常会发生什么

安装脚本运行后，你可能会看到这些提示：

  • 检测系统环境

  • 检测 Git
  • 检测或安装 Homebrew
  • 检测或安装 OpenClaw CLI

只要没有看到明确的报错中断，就继续等它跑完。

Node 版本要不要自己先装

官方当前文档里写的是：

  • Node 24 为推荐版本
  • Node 22.14+ 仍支持

对新手来说，最简单的做法不是自己先折腾 Node，而是让安装脚本自动处理。

你可以把它理解成“把 OpenClaw 真正配置成可用状态”的那一步。

第三步：向导里每一屏怎么选

这里是整篇文章最重要的部分。
界面文字可能会因版本略有不同，但大方向基本一致。

1. 如果出现 Existing config / Keep / Modify / Reset

第一次安装通常不会看到这一步。
如果你看到了，说明这台 Mac 以前装过 OpenClaw 或留下过旧配置。

你可以这样选：

  • 想认真重装一次，选 Reset
  • 想保留原来的配置再调整，选 Modify
  • 不想改现有配置，选 Keep

如果你完全不确定自己以前配过什么，我建议选 Reset，这样后面最省心。

2. 出现 QuickStart vs Advanced

选 QuickStart。

原因很简单：

  • 官方把最适合新手的默认值放在这里
  • 第一次安装没必要一开始就改太多参数
  • 先让本地 Gateway 正常工作，比“完全自定义”更重要

3. 出现 Local or Remote 或 This Mac or Remote

选 Local 或 This Mac。

你现在的目标，是先把这台 Mac 跑通。
不要第一次就选远程，不要一开始就把问题复杂化。

4. 出现 Model / Auth

这是整个流程里最关键的一屏。

你应该选“你现在已经有凭证的那个提供商”。

对大多数新手来说，优先顺序可以这样理解：

  • 你有 OpenAI API Key，就选 OpenAI API key
  • 你有 Anthropic API Key，就选 Anthropic API key
  • 你非常明确自己正在使用 OpenAI Code / Codex 登录体系，再选对应选项
  • 你现在只想先装，不准备立刻聊天，才选 Skip

选完后，向导会让你粘贴密钥。
这时候把你的 API Key 粘进去就可以。

5. 出现 Default model

这一步的原则很简单：

  • 选你账户里当前能用的模型
  • 优先选推荐项
  • 不要第一次就手填自定义模型名

如果你看不懂型号差异，就直接选默认推荐模型。
第一次安装的目标是先跑通，不是先做模型评测。

6. 出现 Workspace

直接保留默认值：

新手第一次不要改。
官方 FAQ 也把它当作默认工作区路径。你后面查配置、看日志、做排错时，默认路径会省掉很多麻烦。

7. 出现 Gateway

这一屏按下面选：

  • Port：18789
  • Bind：loopback 或本机访问
  • Auth mode：token
  • Token storage：默认自动生成
  • Tailscale：off

这里最重要的两点一定要记住：

  • 不要第一次就改成公网绑定
  • 不要关闭认证

官方 Dashboard 文档明确提醒：控制面板本质上是管理入口，不适合直接公开暴露。

8. 出现 Channels

这一屏先全部 Skip。

第一次安装不要接：

  • Telegram
  • Discord
  • WhatsApp
  • 其他聊天渠道

原因不是它们不能配，而是你现在先要确认：

  • OpenClaw 已经装好
  • Gateway 确实在跑
  • 浏览器聊天先成功

先把本地最小闭环跑通，再加渠道，排错会容易很多。

9. 出现 Web Search

如果你已经有搜索服务的 API Key，就按向导填写。
如果没有，直接 Skip。

这一步不会影响你先完成第一轮本地聊天。

10. 出现 Daemon

选 Install。

在 macOS 上，这一步通常会把服务注册成 LaunchAgent。

这一屏基本不用做选择，等它跑完就行。

你看到下面这些信号，通常就说明主流程已经成功：

  • Gateway 启动成功
  • 健康检查通过
  • 没有关键错误中断

选 Recommended 就可以。

如果它追问包管理器，优先选：

13. 出现完成页 Finish / Summary

正常情况下，向导结束时会做两件事：

  • 给你一份安装总结
  • 尝试自动打开浏览器 dashboard

如果浏览器没有自动打开，不用慌，下一步手动执行就行。

第四步：安装完成后，马上做这 5 个验收动作

很多人装完就结束了，但真正重要的是“验收”。

1. 先确认命令已经可用

如果它能正常输出版本号，说明 CLI 至少已经装上了。

2. 确认 Gateway 正在运行

按官方 Getting Started 文档，正常情况下你应该看到 Gateway 在 18789 端口监听。

第六步：怎样才算你已经真的装成功了

对零基础新手来说，最实用的成功标准不是“命令跑完了”，而是下面这 4 条同时成立：

  1. openclaw gateway status 正常
  2. openclaw dashboard 能打开浏览器界面
  3. 你在浏览器里发消息能收到回复
  4. openclaw doctor 没有关键错误

只要这 4 条成立，你就已经完成了一次合格的本地部署。

第七步：第一次跑通后，下一步应该做什么

很多新手装成功后，会马上想做更多事情。
我的建议是按这个顺序继续：

  1. 先熟悉 dashboard 里的基本聊天和设置
  2. 再只接入一个聊天渠道，例如 Telegram
  3. 再考虑技能、自动化、远程访问
  4. 最后再研究 Tailscale、SSH tunnel 或公网方案

原因很简单：每次只增加一个变量，你后面才容易定位问题。

  • 不要把 dashboard 直接暴露到公网
  • 不要还没跑通本地聊天，就先折腾高级配置

你第一次安装的目标只有一个：
